Git

Git이란 분산 버전 관리 시스템임

Git 흐름 순서
    1. Main Branch: 주로 'master' 또는 'main'이라는 이름으로 불리는 기본 브랜치입니다. 프로젝트의 메인 브랜치로, 안정적인 코드가 보관되는 곳입니다.

    2. 개인 Branch 생성: 개인적으로 작업을 진행하기 위해 Main Branch에서 새로운 브랜치를 생성합니다. 이 브랜치는 독립적으로 작업하며, Main Branch에 영향을 주지 않습니다.

    3. Clone: 원격 저장소의 내용을 로컬로 복제합니다. git clone 명령어를 사용하여 원격 저장소의 코드를 로컬 저장소로 가져옵니다.

    4. 수정(변화) 발생: 로컬 저장소에서 작업을 진행하여 코드를 수정하거나 변경 사항을 추가합니다.

    5. Commit: 변경 사항을 로컬 저장소에 커밋합니다. git add 명령어로 수정한 파일을 스테이징 영역에 추가한 후, git commit 명령어로 커밋을 생성합니다.

    6. Push: 로컬 저장소의 커밋을 원격 저장소로 보냅니다. git push 명령어를 사용하여 변경된 내용을 원격 저장소에 업로드합니다.

    7. Pull Request(PR): 작업한 내용을 원격 저장소의 관리자에게 보여주고 병합 요청을 보냅니다. 관리자는 변경 사항을 확인하고 필요한 경우 병합합니다. 이를 통해 여러 사람이 협업하고 코드 리뷰를 진행할 수 있습니다.

    8. Merge: PR이 승인되고 병합이 수행되면, 개인 브랜치에서 작업한 내용이 Main Branch에 통합됩니다. 이를 통해 여러 개발자가 동시에 작업한 내용을 하나로 합치고, 최신 버전의 코드를 유지할 수 있습니다.
